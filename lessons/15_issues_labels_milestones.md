# 第15课：Issues、标签与里程碑
目标：用 Issues 管理任务与缺陷，使用标签与里程碑组织进度。

1. 创建 Issue
   - 在 GitHub 仓库  Issues  New issue
   - 标题简洁，描述包含复现步骤、期望结果与截图（如方块旋转异常）

2. 标签（Labels）
   - 常用：bug、feature、documentation、refactor、good first issue
   - 给 Issue 打标签，便于筛选与统计

3. 里程碑（Milestones）
   - 示例：v0.2 版本、第一迭代
   - 将相关 Issues 关联到里程碑，跟踪完成比例与截止日期

4. 与分支关联
   - 在分支名中包含 Issue 编号：feature/15-hard-drop-opt
   - 提交信息引用：fixes #15 或 closes #15，合并后自动关闭 Issue
